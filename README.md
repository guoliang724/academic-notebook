## 项目简介

这是一个**纯前端单文件**的个人学术英语知识库，专为学习学术写作、法律英语、英译中等场景设计。

所有数据直接嵌入在 `index.html` 文件中，无需后端、无需数据库，可直接部署到 GitHub Pages，也可以本地用浏览器打开使用。

---

## 功能特性

| 功能 | 说明 |
|------|------|
| 📥 **导入语料** | 粘贴 JSON（单条 `{}` 或批量数组 `[...]`）即可新增语料 |
| 🗑️ **删除语料** | 侧边栏 hover 时显示删除按钮，带确认弹窗 |
| ✏️ **行内编辑** | 译文、翻译心法、语法剖析、词汇表均可在页面内直接编辑 |
| 📑 **三大板块** | 译文对比 & 技巧 / 语法句式剖析 / 重点词汇 |
| 🚀 **GitHub 同步** | 一键将最新数据库推送回 GitHub 仓库，永久保存 |
| 📖 **字段说明** | 导入弹窗内置折叠式 JSON 字段文档 |
| 🧮 **MathJax 支持** | 词汇板块支持 LaTeX 数学公式渲染 |
| 🎨 **暗色主题** | 全局深色 UI，带渐变、微动效和自定义滚动条 |

## 快速开始

直接用浏览器打开 `index.html` 即可，无需安装任何依赖：

## 使用指南

### 导入语料

1. 点击左侧侧边栏顶部的 **➕ 导入** 按钮
2. 在弹出的对话框中粘贴 JSON 代码
3. 点击 **💾 解析并存入题库**

支持两种格式：
- **单条**：`{ "title": "...", ... }`
- **批量数组**：`[{ "title": "..." }, { "title": "..." }]`

> ⚠️ **注意**：导入操作仅更新浏览器内存。若需永久保存，请在导入后点击顶部的「🚀 同步 GitHub 仓库」。

---

### 浏览与切换语料

- 点击左侧导航栏中的语料标题即可切换
- 激活的语料以**蓝色左边线 + 渐变背景**高亮显示
- 顶部卡片显示**英文原文**（斜体大字），右上角显示体裁标签

---

### 编辑板块内容

每个内容卡片右上角都有 **✏️ 编辑** 按钮，点击后进入编辑模式：

| 板块 | 可编辑内容 |
|------|-----------|
| 📝 译文 | 直接编辑中文译文文本框 |
| 💡 翻译心法 | 每条技巧单独一个文本框，可 **➕ 添加** 或 **✕ 删除** |
| 🔍 语法剖析 | 每个条目的「名称」「骨架」「说明」分别编辑，可增删条目 |
| 📊 词汇表 | 每行的「词汇」「词性」「释义」「推荐度」分别编辑，可增删行 |

编辑完成后点击 **💾 保存** 立即生效；点击 **取消** 放弃修改。

> 切换到其他语料时，所有未保存的编辑模式会自动取消。

---

### 删除语料

将鼠标 **hover** 到左侧导航栏的语料条目上，右侧会出现 🗑️ 删除按钮（红色，默认隐藏）。点击后弹出确认对话框，确认即删除，并自动切换至相邻语料。

> ⚠️ 同样，删除操作仅影响浏览器内存，需要同步 GitHub 才能永久生效。

---

### GitHub 同步

此功能将当前页面（含最新数据库）推送到你的 GitHub 仓库，实现云端永久保存。

**配置步骤：**

1. 点击顶部 **🚀 同步 GitHub 仓库** 按钮
2. 填入以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| GitHub PAT | 个人访问令牌（需有 `repo` 权限） | `ghp_xxxxxxxxxxxxxxxx` |
| GitHub 用户名 | 你的 GitHub 账户名 | `yourname` |
| 仓库名称 | 存放此文件的仓库名 | `academic-notebook` |
| 文件路径 | 含文件名的路径 | `index.html` |
| 分支名称 | 默认 `main` | `main` |

3. 点击 **💾 保存设置**（配置会存入浏览器 localStorage）
4. 点击 **📤 立即同步推送**

> 推送成功后，GitHub Pages 会在数分钟内自动重新部署。

## JSON 格式规范

### 标准格式（推荐）

这是本笔记本的**内部标准格式**，字段名称与页面一一对应：

```json
{
  "title": "1. 芝加哥学派与摩天大楼",
  "genre": "建筑史学 / 说明文",
  "body": "The career of the Chicago architect, Louis Sullivan, coincided with the so-called Chicago School of Architects...",
  "translation": "芝加哥建筑师路易斯·沙利文的职业生涯，恰逢所谓的"芝加哥建筑学派"兴起……",
  "insights": [
    "动名转换：abundance of steel 译为"钢材的大量供应"更符合工业文体。",
    "顺应中文逻辑：facilitated 译为"得益于"更流畅。"
  ],
  "grammar": [
    {
      "title": "同位语与非限制性定语从句",
      "skeleton": "The career of Sullivan (主) + coincided with (谓) + Chicago School (宾), which...",
      "notes": "which 引导非限制性从句补充学派功能，known as a skyscraper 是过去分词做定语。"
    }
  ],
  "vocab": [
    {
      "word": "coincide with",
      "type": "v.",
      "meaning": "恰逢；重合",
      "rating": "⭐⭐⭐⭐⭐"
    },
    {
      "word": "facilitate",
      "type": "v.",
      "meaning": "推动；使便利",
      "rating": "⭐⭐⭐⭐⭐"
    }
  ],
  "specialHTML": ""
}
```

### 兼容格式（AI 生成）

系统会自动将以下 AI 常用字段名**映射**为内部格式，无需手动修改：

```json
{
  "title": "3. 房屋租赁契约定义",
  "genre": "法律英语 / 合同文本",
  "body": "The lease is a contract to rent by which...",
  "translation": "租赁合同是指……",

  "grammar": "1. 定语从句嵌套：by which 界定合同内容。\n2. 法律命名：-or 施动者与 -ee 受动者对应。",
  "translation_tips": "landlord/tenant 为口语，lessor/lessee 为法律术语。grant 译为'让与/授予'。",
  "takeaway": "模板：[Agreement] is a contract by which [Party A] grants [Right] to [Party B]."
}
```

自动映射规则：

| AI 字段名 | → | 内部字段 | 转换方式 |
|-----------|---|---------|---------|
| `translation_tips` | → | `insights` | 按换行符 `\n` 拆分为数组 |
| `tips` | → | `insights` | 同上 |
| `grammar`（字符串）| → | `grammar`（数组） | 自动包装为含 title/skeleton/notes 的对象 |
| `takeaway` | → | `insights` 末尾 | 追加为「📌 句型模板：...」 |

---

### 字段说明表

| 字段名 | 类型 | 必填 | 含义 / 页面位置 |
|--------|------|------|----------------|
| `title` | `string` | ✅ | 语料标题，显示在左侧导航栏（建议格式：`"序号. 主题"`） |
| `genre` | `string` | 否 | 体裁 / 学科领域标签，显示在原文卡右上角徽章 |
| `body` | `string` | ✅ | 英文原文语料，显示在顶部原文卡（斜体大字） |
| `translation` | `string` | 否 | 精雕中文译文，显示在「译文对比」Tab 第一张卡 |
| `insights` | `string[]` | 否 | 翻译心法列表，每个字符串是一条技巧（显示为列表项） |
| `grammar` | `object[]` | 否 | 语法剖析数组，显示在「语法句式剖析」Tab |
| `grammar[].title` | `string` | — | 语法现象名称（加粗显示） |
| `grammar[].skeleton` | `string` | — | 句型骨架拆解（蓝色等宽字体） |
| `grammar[].notes` | `string` | — | 详细分析说明（灰色小字） |
| `vocab` | `object[]` | 否 | 词汇表数组，显示在「重点词汇」Tab 的表格 |
| `vocab[].word` | `string` | — | 词汇或短语 |
| `vocab[].type` | `string` | — | 词性（`n.` / `v.` / `adj.` 等） |
| `vocab[].meaning` | `string` | — | 中文释义 |
| `vocab[].rating` | `string` | — | 学术推荐度（如 `⭐⭐⭐⭐⭐`） |
| `specialHTML` | `string` | 否 | 自定义 HTML 片段（支持 MathJax 数学公式），渲染在词汇板块底部 |

## 三大板块详解

### 📝 译文对比 & 技巧

**卡片一：精雕学术译文**
- 显示 `translation` 字段内容
- 点击 ✏️ 可直接编辑译文文本

**卡片二：翻译心法拆解**
- 显示 `insights` 数组，每条一个列表项
- 编辑时每条独立 textarea，可增删

---

### 🔍 语法句式剖析

每个 `grammar` 条目渲染为一个卡片，包含：

```
┌────────────────────────────────────┐
│ 同位语与非限制性定语从句  ← title  │
│                                    │
│ The career of Sullivan (主) + ...  │← skeleton（蓝色等宽字）
│                                    │
│ which 引导非限制性从句……           │← notes（灰色小字）
└────────────────────────────────────┘
```

---

### 📊 重点词汇 & 专属拓展

**词汇表**：将 `vocab` 数组渲染为三列表格：

| 词汇 | 词性 / 释义 | 学术推荐度 |
|------|-----------|-----------|
| coincide with | [v.] 恰逢；重合 | ⭐⭐⭐⭐⭐ |

**特殊拓展区**（可选）：若 `specialHTML` 不为空，渲染在表格下方的蓝色卡片中，支持 MathJax 数学公式。

## GitHub Pages 部署

1. Fork 或新建一个 GitHub 仓库
2. 将 `index.html` 上传到仓库根目录
3. 进入仓库 **Settings → Pages**，Source 选择 `Deploy from a branch`，Branch 选择 `main`，目录选 `/ (root)`
4. 几分钟后访问 `https://<用户名>.github.io/<仓库名>/` 即可

**此后更新数据**：在页面内导入新语料后，通过内置的「🚀 同步 GitHub 仓库」功能一键推送，无需手动操作 Git。

## 技术栈

| 技术 | 用途 |
|------|------|
| HTML5 + Vanilla JS | 核心页面与逻辑，零框架依赖 |
| [Tailwind CSS CDN](https://cdn.tailwindcss.com) | UI 样式（CDN 引入，无需构建） |
| [Google Fonts – Inter & JetBrains Mono](https://fonts.google.com) | 正文字体 + 等宽代码字体 |
| [MathJax 3](https://www.mathjax.org/) | LaTeX 数学公式渲染（按需加载） |
| [GitHub REST API](https://docs.github.com/en/rest) | 云端同步推送（PUT /contents） |
| `localStorage` | 本地缓存 GitHub 配置（Token 等） |
